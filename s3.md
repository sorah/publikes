## current: `data/public/current.json`

- `head`: active batch ID (the batch currently receiving new likes)
- `last`: previously closed batch ID
- `updated_at`: unix timestamp of last update

## `batch`: `data/public/batches/{id}.json`

- `id`: batch identifier (timestamp-based)
- `head`: bool — true if this is the active (head) batch
- `pages`: array of page IDs (newest first)
- `next`: next older batch ID (for linked-list traversal), or null
- `created_at`: unix timestamp
- `updated_at`: unix timestamp

## `page`: `data/public/pages/{id}.json`

- `id`: `merged/{batch_id}/{num}` or `head/{batch_id}/{status_id}`
- `statuses`: array of status entries
  - `id`: status (tweet) ID string
  - `ts`: unix timestamp of the like
- `created_at`: unix timestamp

## `status`: `data/private/statuses/{id}.json`

- `id`: status (tweet) ID string
- `complete` (bool): whether data collection is complete
- `graphql_data`: Twitter GraphQL API response (may be null)
- `fxtwitter_data`: fxtwitter API response
  - `tweet`:
      - `url`: tweet URL
      - `id`: tweet ID string
      - `text`: tweet text
      - `created_timestamp`: unix timestamp of the tweet
      - `author`:
        - `id`: author user ID
        - `screen_name`: author handle
        - `name`: display name
      - `media`:
        - `all`: array of media items (photos, videos, gifs)
      - `quote`: quoted tweet (same structure as `tweet`, if present)
- `saved_at`: unix timestamp when the status was saved

## `media index`: `data/private/media/{status_id}/index.json`

- `status_id`: status (tweet) ID string
- `media`: array of media entries
  - `key`: S3 key for the media file (e.g. `data/private/media/{status_id}/photo-1.jpg`)
  - `source_url`: original media URL
  - `filename`: media filename (e.g. `photo-1.jpg`, `video-1.mp4`, `gif-1.mp4`)
  - `filesize`: file size in bytes
  - `original`: original media item from fxtwitter response

## `media file`: `data/private/media/{status_id}/{filename}`

Media files stored with filenames following the pattern `{type}-{num}.{ext}`:
- `photo-{n}.jpg` (or `.png`, `.webp`)
- `video-{n}.mp4`
- `gif-{n}.mp4`
